## D. 3 New Ideas (GeoAI + World Model)

### Idea 1: Event-based World Model for Disaster Triggering & Cascading Risks

**Problem**: Urban/Agricultural areas face disaster (flood, fire, landslide) triggering & cascading risks. Existing prediction is mostly continuous spatiotemporal modeling, lacking joint modeling of sparse events (trigger time) and causal chains (cascading diffusion).

**Approach**:
- Fuse RS FM embeddings (Copernicus-FM/SkySense/OlmoEarth) to capture surface state.
- Inject exogenous drivers: Earth-2/GraphCast/Pangu-Weather meteorological/climate scenarios.
- Event process modeling: Neural STPP / Diffusion STPP (Sparse trigger + Spatiotemporal diffusion).
- Uncertainty calibration: Bayesian Neural Fields (BayesNF).

**Data**:
- Sentinel-1/2, Landsat (Surface state)
- ERA5/Earth-2 (Meteorological drivers)
- OSM (Infrastructure & population exposure)
- Historical disaster event database (emer, GDACS)

**Baseline/Eval**:
- Compare with traditional disaster indices (NDVI anomaly, Fire Weather Index, Flood Inundation Model).
- Metrics: Event lead time, F1@Recall, ECE calibration, CRPS.

**First Experiment Checklist**:
1. Data collection & cleaning (5 data sources, align spatiotemporal resolution).
2. Pre-trained embedding extraction (Fix Copernicus-FM/OlmoEarth).
3. Event labeling & intensity encoding (Historical disaster events).
4. STPP model training & ablation (Embedding vs Embedding+Meteo Driver).
5. Eval & Visualization (Disaster heatmap + Event probability time series).

**Risks**:
- Scarcity of event labels & distribution shift.
- Complexity of multi-source data spatiotemporal alignment.
- Lag & error propagation of exogenous meteorological drivers.

### Idea 2: On-Orbit Active Perception Lightweight World Model Encoder (CubeSat/Edge Distillation)

**Problem**: Existing on-orbit imaging satellites lack real-time decision capability, often capturing cloud cover or low-value areas, leading to low mission efficiency and high downlink load.

**Approach**:
- Distill Spatiotemporal FM (Copernicus-FM/SkySense/OlmoEarth) into lightweight edge encoder (<100MB, <1TOPS).
- Integrate dynamic targeting strategy (NASA Dynamic Targeting concept): Real-time scoring of next scene value (Cloud cover + Event hotspot + Historical value).
- Decision strategy: Cloud avoidance + Hotspot tracking + ROI priority.

**Data**:
- Distillation data: Sentinel-2/Landsat time series + Cloud mask + Event hotspot labels.
- Edge model: Quantized Transformer encoder (8-bit/4-bit).
- Mission goals: Maximize usable image rate, Minimize response latency, Constrain energy budget.

**Baseline/Eval**:
- Compare with random shooting, fixed mission planning, traditional cloud detection scheduling.
- Metrics: Usable image ratio, Hotspot capture rate, Edge inference latency, Energy consumption.

**First Experiment Checklist**:
1. Teacher model selection (SkySense++ vs OlmoEarth vs Copernicus-FM).
2. Distillation strategy design (Feature distillation + Output logits + Temporal contrastive loss).
3. Edge deployment adaptation (TensorRT-Lite / TFLite / ONNX quantized).
4. Simulation environment testing (CubeSat attitude dynamics + Orbit prediction).
5. End-to-end eval (1-week simulation mission vs Baseline).

**Risks**:
- Strict constraints on edge compute & memory.
- Accuracy loss after distillation (especially sparse event detection).
- Robustness of real-time decision (Extreme weather, communication outage).

### Idea 3: Geo-RAG + World Model Agent for Urban Digital Twins (Traffic/Telecom Scenarios)

**Problem**: Urban Digital Twins need to fuse multi-source spatiotemporal data (RS, IoT, Telecom, Traffic) and provide interpretable decision support & counterfactual simulation, but existing platforms lack unified retrieval & causal reasoning capabilities.

**Approach**:
- Retrieval Augmented: STAC/Planetary Computer/Earth Engine + Copernicus-Embed + OSM (GeoLink).
- Knowledge Fusion: Multi-modal embedding alignment (RS + Map + Semantics).
- World Model Simulation: Earth-2/ClimaX/GraphCast scenario driver + Spacetimeformer time series prediction.
- Interpretable Output: Natural language explanation + Counterfactual scenarios ("If green space increases by X%, inundation depth decreases by Y%").

**Use-Cases**:
- Urban waterlogging simulation & green space layout optimization.
- Traffic congestion prediction & adaptive signal control.
- Telecom base station load balancing & emergency coverage.

**Baseline/Eval**:
- Compare with traditional digital twin platforms (No RAG/World Model).
- Metrics: Decision adoption rate, Simulation accuracy (RMSE/MAE), Counterfactual rationality (Expert eval), Response latency.

**First Experiment Checklist**:
1. Data access: STAC catalog + Earth Engine data cube + OSM vector layers.
2. Retrieval system: Vector retrieval (Copernicus-Embed) + Rule filtering (Spatiotemporal constraints).
3. World Model integration: Call Earth-2 API / Local ClimaX inference.
4. RAG Prompt design: Retrieval context + Simulation result + Counterfactual generation.
5. Scenario eval: Select Urban/Traffic/Telecom use-case, complete baseline comparison.

**Risks**:
- Multi-source data quality & consistency (Cloud contamination, spatiotemporal resolution mismatch).
- World Model API latency & cost (Not suitable for real-time scenarios).
- Credibility of counterfactual reasoning (Needs physical constraints or expert validation).
